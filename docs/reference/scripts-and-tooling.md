## Makefile

**File:** `Makefile`

GNU Make targets for common development tasks. Requires `make` (Linux, macOS, WSL).

### Targets

| Target | Command | Purpose |
|--------|---------|---------|
| `install` | `pip install -e ".[dev]"` | Install package with dev dependencies in editable mode |
| `lint` | `ruff check src/ tests/` | Run ruff linter over source and test files |
| `typecheck` | `mypy src/` | Static type checking (strict mode) |
| `test` | `pytest -v` | Run all tests with verbose output |
| `test-cov` | `pytest --cov=src/poly_crawler --cov-report=term-missing` | Tests with per-module coverage report |
| `migrate` | `alembic upgrade head` | Apply all pending database migrations |
| `db-up` | `alembic revision --autogenerate -m "$(msg)"` | Generate a new autogenerated migration. Pass `msg=`: `make db-up msg="add xyz"` |
| `db-down` | `alembic downgrade -1` | Roll back the last migration |
| `run` | `uvicorn poly_crawler.main:app --reload` | Start FastAPI dev server with hot reload |
| `dev` | lint → typecheck → test | Full pre-commit validation pipeline |

### Example workflow

```bash
# First time setup
make install

# Development loop
make dev                           # lint + typecheck + test (fast)
make run                           # start server at http://localhost:8000

# Database changes
make db-up msg="add market_slug"   # autogenerate migration
make migrate                       # apply it
```

## PowerShell dev script

**File:** `scripts/dev.ps1`

PowerShell equivalent of the Makefile for Windows-native development (no `make` required).

### Commands

```powershell
.\scripts\dev.ps1 install          # pip install -e ".[dev]"
.\scripts\dev.ps1 lint             # ruff check
.\scripts\dev.ps1 typecheck        # mypy src/
.\scripts\dev.ps1 test             # pytest -v
.\scripts\dev.ps1 test-cov         # pytest with coverage
.\scripts\dev.ps1 migrate          # alembic upgrade head
.\scripts\dev.ps1 db-down          # alembic downgrade -1
.\scripts\dev.ps1 run              # uvicorn --reload
```

### Environment variable for db-up

```powershell
$env:MSG = "add market_slug"
.\scripts\dev.ps1 db-up
```

## CLI entry point

**File:** `src/poly_crawler/cli.py`

Typer-based CLI providing `seed` and `run` commands.

### Commands

```bash
# Seed a parent wallet (Phase 1)
poly-crawler seed --parent 0xABC...

# Bulk seed from file
poly-crawler seed --from-file parents.txt

# List seeded parents
poly-crawler seed --list

# Ignore a parent
poly-crawler seed --ignore 0xABC...

# Start the dev server
poly-crawler run
```

Equivalently:

```bash
python -m poly_crawler.cli seed --list
```

## Schema validation script

**File:** `scripts/validate_schema.py`

Runs `alembic check` to compare ORM models against the actual database schema. Exits with code 0 if in sync, 1 if drift detected.

### Usage

```bash
python scripts/validate_schema.py
# → Schema is up to date — no drift detected.

# On drift:
# → SCHEMA VALIDATION FAILED (exit code 1)
```

### CI integration

```yaml
# Example GitHub Actions step
- name: Validate DB schema
  run: |
    alembic upgrade head
    python scripts/validate_schema.py
```

## App entry point

**File:** `src/poly_crawler/main.py`

Start the application:

```bash
# Development (auto-reload)
uvicorn poly_crawler.main:app --reload

# Production (bind all interfaces)
uvicorn poly_crawler.main:app --host 0.0.0.0 --port 8000

# With production config
POLY_CONFIG=config/production.yaml uvicorn poly_crawler.main:app --reload
```

### Health check

```
GET /health → {"status": "ok"}
```

### Startup sequence

1. `load_config()` reads defaults → env vars → override file → validated Pydantic model
2. `init_engine(config)` creates async SQLAlchemy engine (pool_size=5, max_overflow=10)
3. FastAPI begins serving requests

### Shutdown

- `close_engine()` disposes the engine, draining all connections
- Cancels any in-flight poll cycles via asyncio task cancellation

## Alembic migrations

**File:** `alembic/versions/0001_initial.py`

Creates all 11 tables: `parents`, `accounts`, `clusters`, `cluster_positions`, `alerts`, `paper_trades`, `sibling_balance_snapshots`, `sessions`, `config_snapshots`, `rpc_logs`, `backtest_runs`.

```bash
# Apply pending migrations
alembic upgrade head

# Roll back one migration
alembic downgrade -1

# Check for schema drift
alembic check

# Generate new migration (auto-detect changes)
alembic revision --autogenerate -m "description of change"
```

## Migration patterns

| Pattern | Command | When to use |
|---------|---------|-------------|
| Apply all | `alembic upgrade head` | After pulling new code |
| Roll back | `alembic downgrade -1` | Revert the last migration |
| Reapply | `alembic downgrade -1 && alembic upgrade head` | Rebuild current migration after editing it |
| Check drift | `alembic check` | Verify ORM matches DB schema |
| New migration | `alembic revision --autogenerate -m "msg"` | After modifying ORM models |
| Fresh DB | Drop + `alembic upgrade head` | Dev/test reset |
